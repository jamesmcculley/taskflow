# Decisions

- **2026-07-20** ‚Äî Checkbox exclusion at three scopes: `#notask` tag per line, `taskflow: false|ignore|off` frontmatter per note, and an excluded-folders setting (prefix match on path segments; changing it triggers a full rescan that also evicts newly excluded files from the index). Exclusion happens before ID assignment, so excluded checkboxes never get `^t-` refs appended.

- **2026-07-19** ‚Äî M7: priority is a standalone `!!`/`!!!` token (a trailing "!" in a title never counts); priority sorts above manual order, and a scheduled time (optional `HH:mm` after ‚è≥'s date) sorts within the day ‚Äî compared by codepoint, not localeCompare, whose collation broke the untimed-last sentinel. Capture's recurrence extraction bounds the phrase with a vocabulary whitelist before rrule validates it, because fromText silently ignores trailing junk. Backdated completions timestamp at noon local on the chosen day. Board view is wide-layout only; dragging between columns rewrites the task under the target heading (the no-heading column inserts above the first heading).

- **2026-07-19** ‚Äî M6: task-level Someday is a plain `#someday` tag (typeable anywhere, incl. mobile) rather than a new emoji token; it's hidden from tag chips and unions with someday-project membership. Tonight is a `üåô` flag; enabling it also schedules today when the task isn't already due/scheduled today. Checklist items are checkboxes nested under a checkbox (through any bullet levels, attached to the root task); they still get block IDs but are no longer independent tasks ‚Äî they don't appear in lists or project progress. After-completion repeats use the `after done` / `after completion` suffix on the rrule text. Selection moved from React context into the zustand store so Quick search can navigate-and-select.

- **2026-07-19** ‚Äî Daily-note journal lines are plain `- ‚úÖ HH:mm ‚Ä¶` list items, not checkboxes, so the indexer never re-ingests them as tasks; a hidden `%%taskId%%` comment makes removal on uncomplete exact. Recurring tasks completed twice a day produce two lines; removal takes the last. Daily sync is on by default (the feature was requested as a default workflow) with a settings toggle.
- **2026-07-19** ‚Äî Pinned filters match open tasks only and AND their criteria; tags use prefix matching for nested tags (`dev` matches `dev/backend`). Filters live in data.json (index-owned UI state, like sort order) ‚Äî not in markdown.

- **2026-07-19** ‚Äî M4: single click now selects a task; opening the note moved to double-click/Enter. Manual drag-reorder writes list-scoped integer order values to the index only ‚Äî markdown line order is untouched, so reordering never causes file churn. Reordering is enabled in manually-ordered lists (Inbox, Today, Whenever, Someday, project sections) and not in date-driven Upcoming. Completion linger is 1.6s.

- **2026-07-19** ‚Äî The indexer reconciles the History with markdown on every (re)index: completions undone outside the plugin (native checkbox in the note, manual edit) are pruned, and closed non-recurring tasks keep only their latest entry. Recurring tasks are exempt (log is their only history). Orphaned entries for deleted lines are kept as history but removable via right-click ‚Üí Remove from History.

- **2026-07-18** ‚Äî `every Nth <weekday>` recurrence shorthand is normalized to rrule's `every month on the Nth <weekday>` (monthly semantics, the common expectation); rrule's own parser rejects the short form. Unparseable recurrence text falls back to a plain completion with a Notice rather than blocking the checkbox. Recurrence text must start with `every` ‚Äî rrule's fromText silently returns a yearly rule for arbitrary junk.
- **2026-07-18** ‚Äî Next occurrence = first strictly after max(anchor, today), with dtstart at the anchor: overdue repeats skip into the future but stay aligned to the original pattern, and completing early still advances past a future anchor.
- **2026-07-18** ‚Äî The History reads only the index completion log (per spec); tasks checked off by hand in markdown don't appear there. Uncompleting removes the task's most recent log entry. Recurring completions exist only in the log, since the line is rewritten as the next occurrence.
- **2026-07-18** ‚Äî Whenever includes tasks that have a due date but no scheduled date (spec says "no scheduled date"; a deadline alone doesn't make a task "scheduled").
- **2026-07-18** ‚Äî Upcoming/Today placement uses the earlier of scheduled/due (same effective-date rule the M2 tabs used). Upcoming labels: Tomorrow, weekday names through the end of the current Monday-start week, then `Jul 25` (with year if different).
- **2026-07-18** ‚Äî Narrow/wide layout switches at 480px container width via ResizeObserver: narrow = nav behind a menu toggle, wide = two-pane.

- **2026-07-18** ‚Äî Plan adjusted: the panel should provide a complete, polished sidebar UI (see ROADMAP.md). Consequences: the M2 This Week/Future tabs are removed in M3 (Upcoming covers both, date-grouped); Areas join the sidebar via `area: <name>` project frontmatter; a new M4 milestone owns visual fidelity and interactions (animations, magic plus, keyboard nav, drag reorder).

- **2026-07-18** ‚Äî Added "This Week" and "Future" tabs (user request, ahead of M3's Upcoming). Definitions: placement date = the earlier of scheduled/due; This Week = dated after today through Sunday (Monday-start weeks); Future = dated beyond that. Today is excluded (it has its own tab) and undated tasks are excluded (they become Whenever/Someday in M3). M3's date-grouped Upcoming still lands as specced.
- **2026-07-18** ‚Äî Mutations live in a `TaskActions` module on the plugin rather than inside the zustand store: the store stays a pure data container (unit-testable without Obsidian), while actions need vault access. Views still funnel every mutation through this one module.
- **2026-07-18** ‚Äî `moveToProject` inserts into the destination before deleting from the source; a crash in between leaves a duplicate rather than a lost task. Moved tasks are normalized to top-level items, and nested child items do not travel with a moved parent (revisit if checklists become first-class).
- **2026-07-18** ‚Äî Cancelling writes no ‚úÖ stamp to the line (cancellations aren't dated); the timestamp is recorded index-only so the History can group cancelled items by day.
- **2026-07-18** ‚Äî Editor commands ("‚Ä¶ task under cursor") are the "sensible default" targets for hotkey-bindable mutations; sidebar tasks use the context menu and checkbox instead.
- **2026-07-18** ‚Äî Capture parse precedence: `!due <date>` first, then `>Project`, then `#tags`, then the first remaining natural-language date (chrono, forward-biased) becomes the scheduled date. Project names resolve exact ‚Üí prefix ‚Üí substring, falling back to Inbox with a note in the preview.

- **2026-07-18** ‚Äî Unknown checkbox statuses (e.g. `- [/]`, `- [>]` from other plugins/themes) are indexed as `todo` rather than skipped, so tasks never silently disappear from views. Only `x`/`X` map to done and `-` to cancelled.
- **2026-07-18** ‚Äî If a task line already carries any block reference (`^whatever`), it is reused as the stable ID instead of appending a second `^t-xxxxxx` ‚Äî a markdown line can only have one block ref, and existing refs are already stable.
- **2026-07-18** ‚Äî Tasks in ordinary notes (no `type: project` frontmatter, not `Inbox.md`, not a daily note) are treated as Inbox items; the spec only distinguishes project notes from everything else, and `project: undefined` already means Inbox in the data model.
- **2026-07-18** ‚Äî Recurrence text (`üîÅ ‚Ä¶`) is captured up to the next metadata emoji, `#tag`, or trailing block ref, since rrule text never contains those tokens.
- **2026-07-18** ‚Äî Tokenizer and store selectors are pure modules with zero `obsidian` imports so unit tests need no API mocking; only the indexer/view layers touch the Obsidian API.
- **2026-07-18** ‚Äî Daily notes are detected by `YYYY-MM-DD` basenames (the Obsidian default); the Daily Notes core-plugin format setting can be wired in later if needed.
